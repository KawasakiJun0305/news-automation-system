# News Automation System - プロジェクト概要

## 📋 プロジェクト名
**News Automation System**（毎朝のニュースダイジェスト自動生成システム）

---

## 🎯 目的

マスメディアの広告だらけなニュースから脱却し、**自分の興味に特化した、品質の高いニュース** を毎朝自動取得・配信するシステムを構築する。

---

## 📊 全体システム図

```
【データソース層】
  ├─ NewsAPI（無料ニュース配信）
  ├─ RSS フィード（日経、NHK など）
  ├─ EDINET（日本企業決算）
  ├─ arXiv（学術論文）
  └─ 将来：Twitter/X, GitHub など
  
          ↓
          
【処理層】
  ├─ 1. データ取得・正規化
  ├─ 2. フィルタリング（スパム除外）
  ├─ 3. スコアリング（重要度計算）
  ├─ 4. 要約生成（Claude/ChatGPT/Gemini）
  ├─ 5. キーワード抽出
  ├─ 6. 重複排除
  └─ 7. ランキング
  
          ↓
          
【出力層】
  ├─ HTML（新聞風、GitHub Pages で公開）
  ├─ Markdown（Git で管理）
  ├─ Notion（データベース化）
  └─ Note（将来：有料マガジン）
  
          ↓
          
【配信】
  └─ iPad Safari で毎朝チェック ✨
```

## 🏗️ システムアーキテクチャ

### 実行環境
- **開発環境**：あなたの Windows PC（開発・テスト用）
- **本番環境**：GitHub Actions（クラウド、毎朝 6 時自動実行）
- **配信先**：GitHub Pages（無料ホスティング）

### 技術スタック
- **言語**：Python 3.9+
- **API**：
  - NewsAPI（ニュース取得）
  - Claude/ChatGPT/Gemini API（テキスト処理）
  - Notion API（データベース連携）
- **VCS**：Git + GitHub
- **CI/CD**：GitHub Actions

## 🔄 実行フロー（毎朝 6 時）

```
【毎朝 6:00】
  ↓
【GitHub Actions が起動】
  ├─ NewsAPI から記事取得
  ├─ RSS フィードから記事取得
  ├─ 複数ソースのデータを正規化
  ├─ フィルタリング（品質チェック）
  ├─ スコアリング（重要度計算）
  ├─ 最適な LLM で要約生成
  │  （AI → Claude、決算 → ChatGPT、科学 → Gemini など）
  ├─ キーワード抽出
  ├─ 重複排除
  ├─ ランキング
  ├─ HTML 生成（新聞風デザイン）
  ├─ Markdown 生成
  └─ Notion に自動投稿
  
  ↓（約 5-15 分で完了）

【GitHub Pages に自動デプロイ】
  ↓
【Notion に自動投稿完了】
  ↓
【あなたが朝イチで iPad で閲覧】
```

## 📈 設計方針

### 1. **拡張性**
- 新しいニュースソース（Twitter、GitHub など）を簡単に追加可能
- マルチ API 対応：どの LLM を使うかは簡単に切り替え可能
- 将来、AWS などより高性能なサーバに移行可能な設計

### 2. **初心者向け**
- Python 初心者でも理解・改造できるレベルのコメント
- 各関数に詳細なドキュメント（docstring）
- トラブル対応ガイド完備

### 3. **品質重視**
- スコアリング機構で「本当に大事なニュース」を抽出
- 複数ソースの正規化で、形式の違いを自動吸収
- キャッシング機構でコスト削減 & 速度向上

### 4. **コスト最適化**
- 無料 API を活用（NewsAPI, RSS）
- マルチ LLM で「簡単な処理は安い API」に自動振り分け
- キャッシング で無駄な API 呼び出しを削減

### 5. **マネタイズ基盤**
- データ蓄積機構で「業界トレンド分析」を生成可能に
- ユーザーカスタマイズに対応
- API 化の基盤を整備（将来の SaaS 化に対応）
