## ⏰ スケジュール概要

### **フェーズごとの期間**

```
【フェーズ 1：要件分析・設計】
期間：1-2 週間
主な成果物：
- 要件定義書
- システム設計書
- テスト計画書
- WBS（このドキュメント）

【フェーズ 2：環境構築 + API 連携】
期間：2-3 週間
対応：ステップ 1-2
成果物：
- Python 環境完成
- NewsAPI / Claude 連携完了
- テスト成功

【フェーズ 3：コア機能開発】
期間：3-4 週間
対応：ステップ 3-4
成果物：
- HTML / Markdown / Notion 出力完成
- スコアリング・フィルタリング完成
- 統合テスト完了

【フェーズ 4：本番環境設定】
期間：1-2 週間
対応：ステップ 5-6
成果物：
- GitHub Actions 設定完了
- 自動実行確認済み
- iPad で閲覧可能

【合計期間】
約 2-3 ヶ月（週 10-20 時間の投下を想定）
```

## ⚠️ リスク管理

| リスク | 影響度 | 対策 |
|--------|--------|------|
| Python 環境構築で詰まる | 高 | セットアップガイド参照 |
| API キー取得に時間がかかる | 中 | 早めに取得開始 |
| API レート制限に引っかかる | 中 | キャッシング・遅延実行 |
| GitHub Actions の設定複雑 | 中 | ドキュメント詳細記載 |
| テストコード書くのが大変 | 低 | テンプレート提供 |
